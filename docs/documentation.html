Documentation Technique & Fonctionnelle - Ezio

Ezio est une application "Local-First" d'audit et de pilotage assisté par IA. Cette documentation couvre le guide fonctionnel pour les utilisateurs, l'installation pour les développeurs et l'architecture refondue.

1. Guide Fonctionnel (Par Écran)

Ezio est conçu pour simplifier et automatiser la saisie d'audits, de formulaires complexes, et la génération de rapports via l'Intelligence Artificielle. Ce guide décrit les fonctions offertes par chaque écran.

1.1 Éditeur de Formulaire / Audit (Accueil)

C'est le cœur de la saisie des données. Il se présente sous la forme d'un tableur dynamique interactif.

  • Création du formulaire : Définition des colonnes libres (Texte, Liste déroulante, Chapitre) via le bouton "Gérer les colonnes".
  • Saisie en grille : Les utilisateurs peuvent ajouter des lignes (Constats, Questions) et les remplir comme dans Excel.
  • Intervention IA (Baguette Magique ✨) : Colonnes dédiées (ia) permettant de générer automatiquement un contenu à partir des autres cellules de la ligne courante.
  • Éditeur de texte enrichi : Les grandes cellules peuvent être éditées via un mini-éditeur Markdown pour une meilleure mise en page, et disposent aussi d'outils de "Traitement IA rapide" (Traduction, Relecture) accessibles via la fenêtre 🛠️.
  • Filtres et Tris : Permet de filtrer l'affichage par chapitre ou sous-chapitre.
Exemple d'utilisation : Un auditeur cyber définit une colonne "Question", une "Constat", et une "Reco. (IA)". Il remplit 50 lignes, clique sur "Générer tout" (ou ✨ par ligne), et l'IA rédige les recommandations adaptées. L'auditeur relit, modifie le texte via l'éditeur enrichi, et le verrouille (🔒).

1.2 Dashboard (Tableau de bord interactif)

Construit automatiquement des graphiques statistiques à partir des données saisies dans l'Audit.

  • Création de Widgets (Graphiques) : Creer des camemberts, barres, etc.
  • Croisement de données matricielles : Chaque graphique permet de croiser deux colonnes de l'audit.
  • Filtres : Chaque graphique peut être restreint à un "Chapitre".
  • Affichages personnalisés : Afficher valeurs brutes ou pourcentages. Option d'inclusion aux rapports.

1.3 Modèles de Rapports

Gère la bibliothèque logicielle des "briques" (Modules) qui serviront à composer les livrables IA.

  • Gestion des Modèles : Créer différents types de modèles finaux (ex: "Synthèse Managériale").
  • Composition : Glisser/déposer des modules IA dans les modèles.
  • Scope : Définir si le module IA lit l'intégralité du formulaire ou un filtre précis.

1.4 Rapports / Livrables

Le moteur d'exécution et de relecture finaux des modèles créés précédemment.

  • Création de Livrable : Instancie un nouveau livrable depuis un Modèle.
  • Génération IA : Génération des textes et graphiques.
  • Éditeur WYSIWYG Global : L'utilisateur peut faire de la mise en page, corriger le texte ou utiliser les "Outils IA" modaux (Traduire, Reformuler).

1.5 Impression (Exportation)

Le centre de compilation physique des données.

  • Injection Word (.docx) : Utilise des fichiers locaux Word comme gabarits et y coule les textes générés et graphiques du Dashboard à l'emplacement précis de {{CONTENU}}, tout en remplaçant dynamiquement les variables {{TITRE}} et {{DATE}} sur l'intégralité du document.
  • Export PowerPoint (.pptx) : Génère des présentations complètes avec une couverture (TITRE), des intercalaires (CHAPITRE) et pagine automatiquement le résultat texte de l'Intelligence Artificielle de slide en slide (SLIDE) sans aucune troncature. Les tableaux disposent de leurs propres diapositives dédiées.
  • Formatage Intelligent des Tableaux : Lors de l'export Word ou PPTX, les moteurs lisent automatiquement les étiquettes de couleur (colorScheme) des cellules de type "Combo" (Liste déroulante) de l'Audit. Les cellules des tableaux exportés prennent alors nativement la couleur d'arrière-plan correspondante, et le contraste du texte s'adapte automatiquement.
  • Rendu Markdown Fidèle (PPTX) : Le système simule la mise en page du texte riche. Les puces (- ou *) sont converties en véritables listes à puces PowerPoint, l'indentation par espaces est respectée, et le texte en gras (**texte**) s'applique proprement au sein des phrases.

1.6 Agents IA (Configuration)

L'espace d'administration technique de l'Intelligence Artificielle.

  • Verrouillage (🔒) & Outils (🛠️) : Protège les modèles ou les transforme en "Macros" (outils rapides pour l'éditeur Markdown).
  • Prompts Systèmes : Définition de l'âme du modèle.

2. Installation en Local

L'application est conçue pour être lancée sans build complexe (No Webpack/Vite requis pour le runtime), mais nécessite un serveur HTTP local pour contourner les restrictions CORS des modules ES6.

Pré-requis : Un navigateur moderne (Chrome, Edge, Firefox) et un environnement d'exécution léger (Python, Node.js ou extension Chrome).

2.1 Démarrage Rapide

Ouvrez un terminal à la racine du projet :

# Avec Python 3
python -m http.server 8000

# Avec Node.js (http-server)
npx http-server .

Accédez ensuite à http://localhost:8000.

2. Architecture & Refactoring (Phase 5)

Le code a été restructuré pour séparer clairement le Cœur (Logiciels de base), les Modules (Logique métier) et l'UI. Des optimisations majeures (Délégation d'événements, Debounce) ont été appliquées en v2.2.1 pour la performance.

2.1 Structure des Fichiers

Répertoire Rôle Fichiers Clés
js/core/ Fondations State.js, Utils.js, DataUtils.js, UIFactory.js
js/modules/ Fonctionnalités app_audit.js, AuditRenderer.js, app_reports.js, ReportsRenderer.js, app_deliveries.js
js/ui/ Interface Navigation.js, Sidebar.js, Modal.js, MarkdownEditor.js
js/api/ Services api_ia.js

2.2 Utilisation des Fonctions Cœur

Voici comment utiliser les nouvelles bibliothèques centrales introduites lors du refactoring.

UIFactory (Toasts & Composants)

Remplacez toutes les alertes natives par des Toasts non-bloquants.

import { UI } from '../core/UIFactory.js';

// Afficher une notification succés
UI.showToast("Export réussi !", "success");

// Afficher une erreur
UI.showToast("Fichier introuvable.", "danger");

// Générer un badge HTML
const badgeHtml = UI.renderBadge("En cours", "#3b82f6");
// <span class="badge" style="...">En cours</span>

DataUtils (Traitement de données pur)

Déléguez les tris et filtres complexes à ce module pour garder les contrôleurs légers.

import { DataUtils } from '../core/DataUtils.js';

const rows = [ ["Chap1", "Non"], ["Chap1", "Oui"] ];
const columns = [ {id:"c1", type:"chapitre"}, {id:"c2", type:"combo"} ];

// Filtrer et trier
const results = DataUtils.processRows(
    rows, 
    columns, 
    { chapter: "Chap1" }, // Filtre actif
    {},                   // Filtres colonnes
    "Oui",                // Recherche texte
    { colIndex: 1, direction: 'asc' } // Tri
);

// results = [{ data: [...], originalIndex: 1 }]

3. Modules Fonctionnels

3.1 Module Audit (Pattern MVC)

Le module Audit applique désormais une séparation stricte :

  • Contrôleur (app_audit.js) : Gère l'état (State), les appels IA et la logique métier.
  • Vue (AuditRenderer.js) : Reçoit un contexte et génère le HTML (incluant le MarkdownEditor en mode compact pour l'édition de texte enrichie ligne par ligne).
// Dans app_audit.js
import { AuditRenderer } from './AuditRenderer.js';

function renderTable() {
    const context = {
        currentForm,
        filters: { ... },
        actions: { handleSort, deleteRow, ... } // Bindings
    };
    AuditRenderer.render(container, context);
}

3.2 Module Livrables (Async/Await & WYSIWYG)

Ce module gère un workflow complexe de génération de documents via IA.

  • Scope : Sélectionne les chapitres à analyser.
  • Context Build : Construit un tableau Markdown des données filtrées.
  • IA Call : Envoie le contexte + prompt au LLM (via api_ia.js).
  • Éditeur WYSIWYG : Le résultat est éditable dans une zone de texte enrichi. La barre d'outils est ciblée (Titres stricts [N] [T3] [T4] [T5], Gras [G], Souligné [S], Listes, Outils IA) pour s'adapter précisément aux niveaux des modèles de livrables finaux. L'éditeur s'étire dynamiquement jusqu'au bas de la fenêtre après génération pour maximiser la zone de lecture (tout en restant redimensionnable). La conversion bidirectionnelle du HTML visuel en Markdown garantit une persistance parfaite (conservation des sauts de ligne naturels, effacement de la graisse Markdown dans les titres, etc). Le module "Triche" visuellement en omettant d'afficher le tableau complet de l'audit pour éviter de polluer l'éditeur, et greffera proprement ce dernier au document complet exclusivement lors de l'Impression ou de la Sauvegarde MD.
  • Outils IA (Baguette Magique ✨) : Via une pop-up (Split-view), le texte peut être envoyé à un Agent configuré en "Outil". Cela permet de scinder la relectures, la traduction ou d'autres traitements de la génération initiale.
// Exemple de flux (simplifié)
async function generateModule(delivery) {
    try {
        const context = await buildContext(delivery.scope, auditData);
        const response = await ApiService.fetchLLM(modelConfig, [
            { role: 'system', content: prompt },
            { role: 'user', content: [instruction, context] }
        ]);
        delivery.result = response;
    } catch (e) {
        UI.showToast("Erreur IA: " + e.message, "danger");
    }
}

3.3 Impression & Export (Module Unifié)

Le système d'impression remplace les anciens exports individuels.

  • Configuration : Le fichier output_config.json définit les modèles disponibles (Word et PPTX) ainsi que toutes les correspondances de styles personnalisés.
  • Word (.docx) : Utilise docx et JSZip pour injecter le contenu formaté Markdown (Titres, Puces, Paragraphes dynamiquement liés à vos styles Word) dans des modèles existants au niveau de la balise {{CONTENU}}.
  • PowerPoint (.pptx) : Utilise PptxGenJS pour créer des présentations natives selon les matrices définies dans output_config.json. Prise en charge experte de **l'Overflow** : le moteur divise automatiquement votre texte Markdown généré par l'IA sur autant de nouvelles diapositives que nécessaire pour empêcher toute coupure, et isole dynamiquement les Tableaux Markdown de contexte.
  • Outil d'Extraction (extract.html) : Utilitaire compagnon glisser-déposer qui rétro-conçoit vos masques .pptx bruts (XML interne) pour en extraire parfaitement le ratio largeur/hauteur réel (en pouces), les marges (en points), et les palettes de couleurs thématiques pour alimenter ezio sans effort.
  • Widgets & Graphiques : Durant l'Impression Word ou PPTX, les graphiques du Dashboard sélectionnés pour l'export sont automatiquement téléchargés en images PNG renommées à la volée. Cela permet de les intégrer d'un simple glisser-déposer au sein de vos rapports.

3.4 Module Rapports (Pattern MVC)

Le module Rapports gère les bibliothèques de modules IA et la composition des rapports. Il applique le pattern MVC :

  • Contrôleur (app_reports.js) : Gère l'état global et les actions logiques (ajouts, suppressions, mouvements).
  • Vue (ReportsRenderer.js) : Construit l'interface HTML complète et centralise l'écoute des actions utilisateur (Event Delegation).

3.5 Module Dashboard

Le tableau de bord permet l'analyse visuelle des données du formulaire :

  • Éditeur Avancé (Modale) : Interface centralisée pour configurer les graphiques avec prévisualisation en temps réel via Modal.js.
  • Croisements Dynamiques Universels : Les axes principaux et secondaires acceptent toutes les colonnes du formulaire (pas uniquement les listes à choix), permettant une extraction dynamique illimitée.
  • DataLabels & Formats : Support du plugin chartjs-plugin-datalabels pour l'affichage des étiquettes (en couleurs contrastées). Choix d'affichage en Valeurs Brutes ou Pourcentages (%), avec mise à l'échelle dynamique des graduations locales (axe X capé à 100%).
